Add authentication system with user-based credits and anti-abuse protection.

IMPORTANT:

* Do NOT break existing app
* Integrate with current analysis + credit system

---

🧩 1. DATABASE SETUP

Use simple database (SQLite or Supabase)

Create User table:

* id
* email (unique)
* password (hashed)
* credits (number)
* plan (free / pro / premium)
* created_at

---

🧩 2. SIGNUP SYSTEM

Create endpoint:

POST /signup

Requirements:

* Accept:
  email, password

* Validate:

  * Email format (valid email only)
  * Password min 6 characters

* Check:

  * If email already exists → return error:
    "User already exists"

* Hash password using bcrypt

* Create user with:
  plan = "free"
  credits = 5

* Return success

---

🧩 3. LOGIN SYSTEM

POST /login

* Verify email + password
* Return auth token (JWT)

---

🧩 4. AUTH MIDDLEWARE

* Protect routes:
  /analyze
  /optimize

* Require valid JWT token

---

🧩 5. CREDIT SYSTEM (USER-BASED)

Before processing request:

* Check user credits

IF credits <= 0:
return error:
"You’ve reached your limit. Upgrade to continue."

---

Deduct credits:

* analyze → -1 credit
* optimize → -2 credits

Update DB

🧩 6. FRONTEND AUTH UI

Add:

* Signup page (/signup)
* Login page (/login)

Forms:

* Email
* Password

🧩 7. SESSION MANAGEMENT

* Store JWT in localStorage
* Attach token to API requests
